More about mwl files

File headers

Each mwl file starts with an text (ascii) header. The first line of a header always reads: %%BEGINHEADER, and the end is marked as: %%ENDHEADER. In between those two tags lines start with % and can contain one of the following:
  • comment (plain text)
  • parameter / value pair separated by a semicolon followed by a space or tab
  • empty line, which starts a new subheader
In the processing chain headers are sometimes concatenated in the same file, separated by an empty line. These are called subheaders. Within a subheader parameter names should be unique, but the same parameter can exist in multiple subheaders. Here is an example of a valid header:
%%BEGINHEADER
% File Format:  feature
% Fields:       timestamp,5,8,1 diode1,5,8,2    diode2,5,8,2    headpos,5,8,2	headdir,5,8,1   speed,5,8,1     movedir,5,8,1
% File type:    Binary
% Program:      Matlab mwlIO toolbox
% Program version:      local
%
% this is another subheader
% Sampling rate: 2000
%
%%ENDHEADER

File types

Right after the header in a mwl file the data starts. The data can either be stored in ascii or binary format. There are several different file types which differ in the way the data is stored:
  • Boundary file. In this file type cluster boundaries are stored. A boundary is defined for a cluster in a particular 2-dimensional projection and consists of a list coordinates.
  • Record files. Most files fall in this category. Data consists of a list of records, each containing data in a set of fields. The fields are described in the 'Fields' parameter in the header. There are two kinds of record files:
    • Variable length records.
      • Raw position file.
    • Fixed length records.
      • Diode file.
      • EEG file.
      • Event string file.
      • Feature file.
      • Waveform file.
